## FFA 血管提取复现计划（无标注、可落地版）

### Summary
目标是在你的 `512×512` 灰度 FFA 私有数据上，复现论文的**无训练**多尺度形态学血管分割流程，并交付 `Notebook + CLI` 两套实现。  
成功标准采用“双轨验收”：`可视化人工抽检 + 无监督统计门禁`。数据划分不用于训练，仅用于管理与结果分组；主流程按你选择对全量数据统一推理。

### Key Changes / Implementation Changes
1. 核心算法流水线（与论文对齐，先不启用 ROI）
   - 输入灰度图 `I0`。
   - 两尺度金字塔：`s0=原图`，`s1=下采样2倍`。
   - 光照校正：`Ieq = Is / (mean_filter(Is, N) + c)`。
   - 多方向改进 top-hat：`M=9` 个线结构元素（`0°~180°` 均匀分布）。
   - 方向响应融合：逐像素取 `max`。
   - 自适应阈值二值化（局部均值型）。
   - 面积开运算去小连通域（阈值可配）。
   - 跨尺度融合：上采样回原尺寸后做像素 OR，得最终血管 mask。

2. 参数策略（按 512 分辨率比例缩放）
   - 论文基准：`L={30,15}`, `N={35,30}`, `area_min=200`（对应 3900×3072）。
   - 第一版初始化（按短边比例约 `512/3072≈0.167` 缩放并设下限）：
     - `L0≈5~7`, `L1≈3~4`
     - `N0≈7`, `N1≈5`
     - `area_min≈20~40`
   - 采用“小步人工微调”而非大网格搜索：先固定 `M=9` 与双尺度，优先调 `N`、`L`、`area_min`。

3. 工程交付
   - `Notebook`：用于单图可视化调参，展示中间结果（`Ieq`、top-hat 响应、阈值图、后处理图、最终叠加）。
   - `CLI`：目录级批处理，支持输入 `train/val/test` 或合并目录，统一推理并导出结果。
   - 每图输出：`mask`、`overlay`、关键中间图；并生成汇总 `CSV`（无监督统计）。

4. 接口与产物约定
   - CLI 示例：
     - `python run_infer.py --input_dir ... --output_dir ... --save_intermediate --config config.yaml`
   - 配置项（`yaml/json`）：
     - `scales, M, L_per_scale, N_per_scale, c, adaptive_block_size, adaptive_C, area_min`
   - 输出目录结构：
     - `masks/`, `overlays/`, `intermediate/`, `metrics_unsup.csv`, `run_config_snapshot.yaml`

### Test Plan
1. 功能正确性
   - 单图 smoke test：各阶段输出尺寸、类型、取值范围正确。
   - 批处理稳定性：100+ 图连续运行无崩溃，异常图可跳过并记录日志。

2. 可视化验收（人工抽检）
   - 每个子集随机抽样（如各 20 张）查看 `overlay`：
     - 主干血管连续性
     - 细血管可见性
     - 背景噪点与误检水平
     - 边缘区域误检情况（因暂不做 ROI 重点观察）

3. 无监督统计门禁
   - 每图统计：血管面积率、连通域数、最大连通域占比、骨架总长度、平均分支度。
   - 以分位数/IQR 规则标记异常样本，输出待复核清单。

4. 参数收敛流程
   - 在固定小样本集上迭代 2~3 轮：
     - 先调 `N`（光照平滑尺度）
     - 再调 `L`（血管宽度匹配）
     - 最后调 `area_min`（噪点抑制）
   - 锁定参数后全量推理一次并固化配置。

### Assumptions
- 当前任务不做监督训练与论文指标（Pr/Rc/Sp 等）对齐，因为无 GT。
- `train/val/test` 仅作数据管理分组，不承担训练/验证职责。
- 第一版暂不做论文 ROI 椭圆裁剪；若边缘误检明显，第二阶段再加入可开关 ROI 做 A/B。
- 默认血管在你的 FFA 灰度图中表现为相对亮结构；若存在对比反转，将在预处理加入可选强度反相开关。
